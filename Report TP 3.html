How to handle deletion:

There are ways to delete a referenced table:

  1. Drop Foreign Key Constraints (Risky): We can drop the foreign key constraints referencing the table we want to delete. However, this weakens data integrity and allows orphaned rows(rows pointing to non-existent data) to exist.
  2. Delete Referencing Rows First: We can delete all rows in the referencing tables that reference the rows in the table you want to delete. This ensures no orphaned rows remain.
  3. Cascade Deletes (Carefully): MySQL allow setting ON DELETE CASCADE when defining foreign key constraints. This automatically deletes referencing rows in the referencing table when a row is deleted in the referenced table. Use this option cautiously, as unintended data loss can occur.
